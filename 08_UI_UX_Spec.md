# Mehndi Showcase & Booking Platform

## UI / UX Design Specification

Version: 1.0

Design Type: Premium Mobile-First Luxury Portfolio Platform

Primary Device Target:

Mobile

Secondary Device Targets:

Tablet

Desktop


# 1. Design Philosophy

The platform must communicate:

Elegance

Trust

Professionalism

Luxury

Simplicity

The design must feel premium while keeping the focus on mehndi artwork.

Designs are the hero.

The interface is supporting content.


# 2. UX Principles

## Mobile First

Every screen must be designed for mobile first.

Desktop adapts from mobile.


## Conversion First

Every screen should encourage:

Design Discovery

WhatsApp Contact

Booking Submission


## Content First

Photography should always receive visual priority.

UI should never compete with images.


## Fast Interaction

Maximum:

3 taps to booking
1 tap to WhatsApp

# 28. Design Do’s

Prioritize photography

Use whitespace generously

Keep CTA visible

Maintain hierarchy

Optimize for thumb usage

Keep navigation predictable


# 29. Design Don’ts

No cluttered layouts

No hidden booking actions

No autoplay videos

No excessive colors

No intrusive popups

No tiny touch targets


# 30. Conversion Optimization Rules

Every page must include at least one:

Book Now CTA

WhatsApp CTA


Design Detail Pages must include:

Sticky Book Button

Sticky WhatsApp Button


Contact options must never require more than:

2 taps


# 31. Lighthouse Targets

Mobile:

Performance ≥ 95

Accessibility ≥ 95

Best Practices = 100

SEO = 100


# 32. UI Acceptance Criteria

Users must be able to:

Find a design within 10 seconds

Reach booking within 3 taps

Contact artist within 1 tap

Complete booking in under 60 seconds

Browse comfortably on mobile devices

The platform must maintain a premium luxury appearance while remaining fast, accessible, and conversion-focused across all supported devices.
